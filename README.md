# Blog Workspace

ブログコンテンツ作成ツール。AIが下書きを生成し、人間が編集・仕上げまでを一気通貫で行う4ペイン型ワークスペース。

## 概要

ソース素材（テキスト・PDF・URL・音声など）をアップロードすると、AIが見出しと本文の下書きを生成。生成された内容をエディターで編集し、Markdownで書き出すまでの作業を1画面で完結させる。

`workspace-ui-kit`（同リポジトリ内）のペイン構造・UIコンポーネント群を参考に、新規リポジトリとして構築する。

---

## ペイン構成

```
┌─ Pane 1 ──┬─ Pane 2 ──┬─────── Pane 3 ───────┬─ Pane 4 ─┐
│           │           │                       │          │
│  記事一覧  │  見出し   │      エディター        │  画像    │
│           │           │                       │          │
│  公開済み  │  H1/H2/H3 │   Tiptap             │  アップ  │
│  草稿      │  一覧     │   WYSIWYG            │  ロード  │
│           │  (目次ナビ)│                       │  ＋生成  │
└───────────┴───────────┴───────────────────────┴──────────┘
```

### Pane 1: 記事一覧

- 公開済み・草稿を含む全記事を表示
- ステータス（下書き / レビュー中 / 公開済みなど）を表示

### Pane 2: 見出し

- AI生成された見出し（H1/H2/H3）の一覧
- クリックすると Pane 3 の対応セクションへスクロール（目次ナビ）
- 見出しの追加・並び替え・削除も可能

### Pane 3: エディター

- **Tiptap**（WYSIWYG リッチテキストエディター）を採用
- AI生成の下書きを初期値として読み込み、人間の手で自由に編集
- ソースをアップロードすると AI がセクションごとの本文を生成

### Pane 4: 画像

- アイキャッチ画像・セクション口絵の管理（本文内インライン挿入は行わない）
- アップロード（ローカルファイル選択）
- AI生成（プロンプト入力 → モック画像を返す）

## グローバルヘッダー

| ボタン | 機能 |
|--------|------|
| ソースアップロード | 記事ごとに素材を紐付け。AI下書き生成のトリガー |
| 保存 | 現在の記事を保存 |
| 書き出し | Markdown (.md) 形式でダウンロード |

---

## ソースの種類

アップロードできる素材は以下の5種類。実装フェーズ1ではUIのみ実装し、AI連携部分はモックアダプタで差し替え可能な構造にする。

| 種類 | 例 |
|------|----|
| テキスト / メモ | `.txt`・フォーム直接入力 |
| PDF | 取材資料・レポート |
| URL | 参考記事・ウェブページ |
| Word / Markdown | `.docx` / `.md` |
| 音声・文字起こし | 録音ファイルの文字起こしテキスト |

---

## データ・状態管理

- **Zustand + localStorage** で管理（ブラウザ内永続化）
- 後からバックエンド（Supabase 等）へ差し替えられるアダプタ構造で設計する
- 記事データ・ソース・画像はすべて記事 ID に紐付いて管理

---

## AI連携方針

- **フェーズ1はUIのみ**。AI呼び出し部分はモック関数で固定レスポンスを返す
- 実際の API は後からアダプタを差し替えて接続（OpenAI / Claude 等）
- モックが返す内容:
  - ソースアップロード後 → 見出しリスト＋各セクションの本文テキスト
  - 画像生成プロンプト送信後 → プレースホルダー画像
